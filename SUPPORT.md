# Support

Use the right repository so issues stay easy to triage.

## Use This Repository For

- Reproducible AJ Tools defects in commands, dialogs, or workflows
- Feature requests for add-in behavior, packaging logic, or developer tooling
- Source-maintenance questions tied to the codebase

Before opening an issue:

- include the AJ Tools version, Revit version, and clear reproduction steps
- confirm the problem is not only an installer or download issue
- remove customer names, local paths, and private model data from attachments

## Use AJ-Tools-Installer For

- release downloads
- checksum verification problems
- install, uninstall, or upgrade issues
- current-user versus all-users installer questions

Open installer-facing issues in [AJ-Tools-Installer](https://github.com/Ajmalpshaik/AJ-Tools-Installer/issues).

## Security

Do not post security issues, secrets, or customer data publicly.
Follow [SECURITY.md](SECURITY.md) for private reporting instructions.
